TS Console Object Documentation Classes
TSDoc Script Objects
TSDoc functionality is exposed to script via the SysDoc family of classes. This is accomplished through four main script classes.
SysDocObject: Utility/factory class for working with the family of SysDoc classes. Acts as a general documentation class loader, templating utility factory, etc.
SysDocClass: Class-level documentation object. Typically the objects representing fields and methods will be interacted with through this class.
SysDocField: Field-level documentation object .
SysDocMethod: Method-level documentation object
$SysDoc Global Documenting Engine
On game launch, consoleDoc.tscript creates a global SysDocObject called SysDoc (with global $SysDoc). This object serves as a general documentation class utility factory.
.
SysDocClass Class
The SysDocClass object provides all the available information about an object's class. A SysDocClass object can be generated from any object in a game using:
myDClassObj = SysDoc.getDocClass(Object,FilterParents);
ClassDoc SimGroup Structure
A full class documentation structure can be manifest in script with the ClassDoc SimGroup structure, returned by:
myDClassStruct = SysDocClass.getClassDoc();
This simgroup holds class fields along with child simgroups with field and method objects.
